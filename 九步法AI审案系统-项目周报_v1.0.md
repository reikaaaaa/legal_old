## 一、项目概述

基于最高人民法院"要件审判九步法"方法论与五层法律标签库（L1-L5），我们构建了一套**完整的 AI 智能审案工作流系统**。该系统以大型语言模型（LLM）为核心推理引擎，结合结构化法律知识库，实现从"当事人诉讼材料输入"到"法院裁判文书骨架输出"的全链路自动化审判推理。

系统采用"材料规范层 → 九步法推理引擎 → 保底裁判门 → 裁判文书生成"四阶段流水线架构，同时提供可交互的 Web 前端展示界面与开发者调试检阅器，满足演示、交付与迭代优化三类场景需求。

## 四、核心技术亮点

### 4.1 材料规范层 — 口语化输入直接可用

传统法律 AI 系统要求严格的 JSON 结构输入。本系统独创**材料规范层**，支持当事人使用自然口语描述案情，系统自动完成：
- 案件类型推断与案由匹配
- 当事人/诉讼请求/事实主张/证据的结构化抽取
- **双维度审核**：案由专项（6大核心材料检查项）+ 九步法步骤级（9步逐一预判材料充足度）
- 缺失材料自动识别 + **补正指引生成**（含法律依据、获取路径、优先级排序）

### 4.2 五层法律标签检索体系

构建 L1-L5 五级法律标签索引（法律部门 → 法律领域 → 法律制度 → 法条群 → 规则单元），实现从案件事实到具体法条的**精准语义匹配**，单次检索返回 TOP-12 候选法条，附带 relevancy score。

### 4.3 保底裁判门机制

借鉴自动驾驶分级理念，设计四级风险管控：
- **强裁判**（评分 ≥80）：完整输出，无保留
- **中风险裁判**（60-79）：输出完整结果 + 风险提示标注
- **用户选择门**（40-59）：暂停裁判，呈现选项供用户决定
- **硬阻断**（<40）：仅输出材料缺口清单，不生成裁判

### 4.4 多层容错体系

- **LLM 调用层**：最长 3 次自动重试 + 指数退避
- **超时降级层**：DashScope 调用超时自动关闭 thinking → 换用 qwen-plus → 返回降级标记
- **JSON 解析层**：5 级降级容错（直接解析 → 提取代码块 → 修复尾部截断 → 保底默认值 → 错误标记）
- **步骤异常层**：单步异常不阻断整体流程，缺漏步骤在最终输出中透明标注

### 4.5 Demo 模式与演示就绪

前端内嵌完整预设案例（侵害计算机软件著作权纠纷，含全部 9 步推理结果），**无需启动后端即可零延时展示完整端到端流程**，适合向甲方进行即时演示。同时支持"真实分析"模式，调用后端 LLM 进行实时推理。
