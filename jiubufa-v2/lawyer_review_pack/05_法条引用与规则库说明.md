# 05 · 法条引用与规则库说明

---

## 本层定位

法条引用与规则库是系统的**法律知识基础**。系统通过五层法律标签体系组织法条，在九步法的 Step 2（请求权基础）、Step 3（抗辩权基础）、Step 9（要件归入裁判）三个节点进行法条检索和引用。

---

## 法条库概况

| 属性 | 数值 |
|------|------|
| 法条总量 | 4,559 条标注法条 |
| 来源 | articles_annotated.jsonl（结构化标注） |
| 组织方式 | 五层法律标签体系（L1-L5） |
| 联网补充 | Step 2、Step 3、Step 9 可联网搜索最新法条 |

### 五层标签体系结构

```
L1 法律部门    →  民法 / 刑法 / 行政法 / 商法 / 知识产权法 ...
    │
L2 法律领域    →  合同 / 侵权 / 物权 / 婚姻家事 / 公司 / 劳动 ...
    │
L3 法律制度    →  违约责任 / 侵权责任 / 买卖合同 / 离婚 ...
    │
L4 法条群      →  一般规定 / 损害赔偿 / 免责事由 / 诉讼时效 ...
    │
L5 规则单元    →  民法典_577_001（违约责任一般条款）
                 著作权法_53_001（停止侵害民事责任）
                 民诉法_122_001（起诉条件）...
```

## 法条检索机制

### 检索方式

| 方式 | 说明 | 适用步骤 |
|------|------|---------|
| **五层标签检索** | 基于案由、请求类型、抗辩类型等标签加权打分，返回 TOP-12 候选法条 | Step 2, Step 3 |
| **联网搜索** | 调用搜索引擎实时补充不在知识库中的法条，返回 TOP-2 | Step 2, Step 3, Step 9 |
| **ID精确引用** | 前序步骤已确定的法条ID，直接引用原文 | Step 4, Step 5, Step 9 |

### 检索权重

| 优先规则 | 说明 |
|---------|------|
| 具体规则 > 原则规则 | 《消法》第55条（三倍赔偿）优先于《民法典》第577条（违约责任一般条款） |
| 特别规则 > 一般规则 | 《劳动合同法》优先于《民法典》雇佣合同规定 |
| request_basis > 定义/程序/证据规则 | 直接产生法律效果的规则优先 |
| 新法 > 旧法 | 联网搜索获取的新法条优先于知识库旧版本 |

## 法条引用在系统中的流转

```
Step 2: 检索请求权基础
  └── 输出: rule_unit_id, law_name, article_no, rule_unit_text, relevance_score

Step 3: 检索抗辩权基础
  └── 输出: rule_unit_id, law_name, article_no, rule_unit_text（如有抗辩）

Step 4: 拆解构成要件
  └── 输入: Step2/Step3 确定的法条
  └── 处理: 分析法条结构 → 拆解为要件要素

Step 9: 要件归入并作出裁判
  └── 处理后输出: cited_rules[]（法条ID列表）
  └── 最终通过 KB 解析为: {law_name, article_no, rule_unit_text}
```

---

## 输出法条引用格式

每条引用的法条最终呈现为：

```json
{
  "rule_unit_id": "著作权法_53_001",
  "law_name": "中华人民共和国著作权法",
  "article_no": "第五十三条",
  "rule_unit_text": "未经著作权人许可，复制、发行、表演、放映、广播、汇编、通过信息网络向公众传播其作品的，应当根据情况，承担停止侵害、消除影响、赔礼道歉、赔偿损失等民事责任。"
}
```

## 需要律师重点审查的问题

### 法条准确性

| # | 问题 | 审查要点 |
|---|------|---------|
| 1 | 系统引用的法条是否**真实存在**？ | 有无编造不存在的法条编号？ |
| 2 | 法条编号、名称、文本是否与现行有效版本一致？ | 注意法律修订后的条文变动 |
| 3 | 法条文本是否**完整**？有无截断关键但书或例外条款？ | 部分法条有多款多项，系统是否全部引用？ |

### 法条适用性

| # | 问题 | 审查要点 |
|---|------|---------|
| 4 | 引用的法条是否**真正适用于**该案件事实？ | 有无"看起来相关但实质不适用"的情况？ |
| 5 | 是否存在**程序法、实体法、证据规则混用**？ | 例如在说理部分引用程序法条文 |
| 6 | 是否存在**漏引**重要法条？ | 如知产案件未引用《计算机软件保护条例》 |
| 7 | 是否存在**错引**法条？ | 引用了已被废止或修订的版本 |
| 8 | "特别法优于一般法"的判断是否正确？ | 系统是否正确识别特别法与一般法的关系？ |

### 法条与事实匹配

| # | 问题 | 审查要点 |
|---|------|---------|
| 9 | 每个法条要件是否有对应的**已认定事实**支撑？ | 不能出现"引用法条但没有事实认定基础"的情况 |
| 10 | 裁判结论是否确实**由被引法条所支持**？ | 法条→要件→事实→结论的逻辑链是否完整？ |

## 系统边界

| 系统不做 | 原因 |
|---------|------|
| 判断法条是否违宪 | 不涉及宪法审查 |
| 判断法条之间的效力层级冲突 | 不做立法法层面的效力判断 |
| 引用司法解释的具体条文 | 知识库以法律条文为主，司法解释覆盖不全 |
| 引用地方性法规或行业规范 | 当前知识库仅覆盖全国性法律 |
| 自行判断法条是否已被废止 | 依赖知识库版本和联网搜索获取的信息 |
