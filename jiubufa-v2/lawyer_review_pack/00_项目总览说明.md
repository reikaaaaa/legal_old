# 九步法 AI 审案系统 — 律师审查说明书 · 项目总览

## 一、系统定位

本系统是基于"**要件审判九步法**"（邹碧华法官方法论）与**五层法律标签库**构建的 AI 智能审案辅助工具。系统接收当事人以口语化方式提交的案件材料，自动完成从**材料审核 → 法律要素结构化抽取 → 九步法推理 → 裁判文书框架生成**的全流程分析。

> **核心原则**：系统定位为法官/律师的**辅助分析工具**，不替代法官裁判，不替代律师代理判断。所有输出结论均需法律专业人员审核确认。

## 二、系统分层架构

```
用户原始材料（口语化文本）
        │
        ▼
┌───────────────────────────────┐
│  第 1 层：材料规范层           │  ← 本次审查重点
│  · 案由识别                    │
│  · 材料完整性审核（双维度）     │
│  · 法律要素结构化抽取           │
└───────────────┬───────────────┘
                │ 结构化案件数据
        ▼
┌───────────────────────────────┐
│  第 2 层：九步法推理引擎        │  ← 本次审查重点
│  · Step1-9 完整审判推理链      │
│  · 法条检索 + 联网搜索增强     │
│  · 保底裁判门（四级风险管控）   │
└───────────────┬───────────────┘
                │ 裁判文书框架
        ▼
┌───────────────────────────────┐
│  第 3 层：输出生成层            │  ← 本次审查重点
│  · 裁判文书骨架                │
│  · 一致性校验                  │
│  · 法条引用清单                │
└───────────────────────────────┘
```

## 三、本次律师审查的目标

**不是审代码，是审法律逻辑。** 律师需要判断：

| 审查维度 | 核心问题 |
|---------|---------|
| 材料审核逻辑 | 系统对"材料是否齐全"的判断是否符合立案审查标准？ |
| 案由识别准确性 | 案由识别错误是否会导致后续分析路径全部偏移？ |
| 诉讼请求拆分 | 是否准确识别、拆分、规范化了当事人的原始诉请？ |
| 事实与证据对应 | 事实主张与证据的关联是否符合证据规则？ |
| 否认/抗辩区分 | 系统对"否认""抗辩""抗辩权""程序异议"的分类是否准确？ |
| 举证责任分配 | 举证责任分配是否符合"谁主张谁举证"基本原则？ |
| 法条引用 | 引用的法条是否准确、完整？有无错引、漏引、编造？ |
| 裁判说理 | 推理逻辑是否符合三段论结构？结论是否越权？ |

## 五、系统边界声明

以下事项系统**不做**，由法律专业人员负责：

1. **不做最终法律判断** — 系统输出为"辅助参考"，不构成法律意见
2. **不编造不存在的事实** — 所有事实元素必须来源于用户输入材料
3. **不编造不存在的法条** — 法条引用仅限于已加载的4559条标注法条
4. **不判断证据真实性** — 系统仅标注"待核实"，不对证据三性做终局判断
5. **不替代举证责任分配** — 基于"谁主张谁举证"原则做提示，不替代法官裁量
6. **不做价值判断** — 不涉及舆论、道德、政策考量
